# Design Policy

## 全体デザイン

### 基本方針
- シンプルなテキストベースUI
- vimライクな左寄せレイアウト
- 絵文字は使用せず、ASCIIベースのシンプルなUI
- 全画面モード（AltScreen）でターミナル全体を使用

### レイアウト構造
- ヘッダー: アプリケーション名（グレー）
- コンテンツエリア: 画面の主要コンテンツ
- ステータスエリア: 接続状態などのステータス表示（1行、上下にボーダー）
- フッター: ヘルプテキスト

### マージン
- 全体的に左右に1文字分のマージンを統一（ヘッダー、コンテンツ、フッターすべて）
- ステータスエリアはマージンなし

## UIコンポーネント

### テキスト入力フィールド

長い文字列を入力した場合、表示領域の右端にスクロールし、左側を省略表示します：

```
入力中:       [ ...some-long-endpoint_ ]
フォーカス外: [ ocid1.compartment.oc1.. ]
```

仕様:
- カーソル位置（`_`）は常に表示領域内に保たれます
- フォーカスが外れた時は先頭から表示されます
- `←` / `→` でカーソル移動可能（表示領域も自動スクロール）

### チェックボックス

```
フォーカスなし: [ ] HTTPS/TLS
フォーカスあり: [x] HTTPS/TLS
```

### ラジオボタン

```
選択中:   (*) OCI Config Profile
未選択:   ( ) Instance Principal
```

### ボタン

```
フォーカスなし:   Test Connection
フォーカスあり: > Test Connection
```

## キーバインド

### 共通操作
- `Tab` / `Shift+Tab`: フィールド間の移動（ループ）
- `Ctrl+C`: アプリケーション終了

### エディション選択画面
- `Tab` / `Shift+Tab` または `↑` / `↓`: 選択肢の移動（ループ）
- `Enter`: 決定
- `q`: 終了

### 接続設定画面
- `Tab` / `Shift+Tab`: フィールド間の移動（ループ）
- `Space`: チェックボックス、ラジオボタンの切り替え
- `Backspace`: テキスト入力の削除
- `←` / `→`: テキスト入力フィールド内のカーソル移動
- 文字キー: テキスト入力
- `Enter`: ボタン実行（接続テスト/接続する）
- `Esc`: エディション選択画面に戻る

### テーブル一覧画面
- `j` / `k` または `↑` / `↓`: テーブルリストの上下移動
- `Enter`: 選択したテーブルのデータを表示（将来実装）
- `Esc`: 接続設定画面に戻る
- `q`: アプリケーション終了

## カラーテーマ

- **アクセントカラー**: シアン (#00D9FF) - ボーダー、選択中の要素
- **成功**: グリーン (#00FF00) - 接続成功メッセージ
- **エラー**: レッド (#FF0000) - 接続失敗メッセージ
- **背景**: ダークグレー (#1A1A2E) - ヘッダー、ステータスバー
- **テキスト**: ホワイト (#FFFFFF) - サブタイトル、通常のテキスト
- **グレー**: (#888888) - ヘッダー、ラベル、ヘルプテキスト

## フォームバリデーション

現在の実装: バリデーションなし（将来実装予定）

## 接続状態の表示

### 接続テスト中
ステータス領域に "Connecting..." と緑色で表示

### 接続成功時
ステータス領域に "Connected" と緑色で表示

### 接続失敗時
ステータス領域に "Connection failed" または詳細なエラーメッセージを赤色で表示（画面幅に応じて切り詰め）
